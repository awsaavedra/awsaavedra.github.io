# Writing-Draft

`writing` owns the rules of the prose; this owns the author-first loop that produces it. **The voice answer is structural** — a draft born in the machine's register stays there, and no downstream rule recovers it (`writing`'s voice rule is preserve-only for that reason) — so this skill never writes the first version of anything: it responds to what the author wrote.

## The loop
Outline (author) → angles (skill) → draft (author) → expansion / trim rounds (skill proposes, author disposes) → hand off to `writing` §Revision-Protocol. **Every skill stage is opt-in** — the author enters wherever the piece is (an outline wanting angles, a finished draft wanting expansion or trim) and may skip any stage; fixed is the order of authorship, not the itinerary. `writing`'s rules bind throughout; this file adds only the division of labor.

## Outline — the author's first crack
Respond to the outline; don't replace it.
- **Run `writing` §Thesis here** — the outline is the cheapest place a thesis can fail the worth-writing test; killing it costs a page, not a week.
- **Angles, not edits:** the missing argument · the strongest objection unplanned-for · the ordering the outline didn't consider · the assumption it builds on but never states. Name each and where it would go; the author reorders.
- **Venue named now** — venue decides register, length ceiling, and what the reader already knows (`writing` §Audience walks the rest).

## Development — after the author finishes a draft
- **Expansions are angles, not prose.** Name the missing argument, objection, or concrete instance and where it goes; the author writes it in their own voice. Scaffolding prose only on request, marked as scaffolding — it exists to be rewritten, not kept.
- **Trims may be applied directly** — a cut can't introduce house style. Every trim cites the rule it enforces: `writing` §Thesis minimum-message-length · a digression that never earned itself · an unfired Chekhov's gun.
- **Expand and trim in separate rounds** — generation and criticism are different modes (`writing` "draft ugly, revise cold"); a round that does both stalls both.
- **Converged when neither moves** — no expansion worth its words, no trim that survives the lossless test (§Thesis's ants/Occam stopping rule, applied to rounds). Then hand off: `writing` §Revision-Protocol, type and edit depth named.

## Output
```
## Draft development: <piece> (<type> · <venue> · round <n>)
Stage: outline | draft — <expand | trim>
Angles: <missing argument / objection / instance — where each goes | none>
Trims: <n cuts · <words> removed — rule each enforces | none this round>
Verdict: another round (<expand | trim>) | converged → §Revision-Protocol (<type> · <depth>)
```

## Gates
- Author wrote the first version of every artifact on the page — outline, draft, every expansion's final wording.
- Expansions delivered as angle + location; any scaffolding prose explicitly marked.
- Every trim cites the rule it enforces.
- Expand and trim rounds never interleaved.
- Hand-off names type and edit depth (`writing` §Revision-Protocol).
